Advanced Configuration Guide

This scanner runs inside the browser, so the strongest results come from using the right mode at the right time. Use this quick brochure when you need better accuracy without exposing secrets in public code.

1

Run A Direct Scan First

Start with a normal scan to see what any visitor can reach. This gives you the public attack surface before you add authentication or a relay proxy.

2

Add Auth Only When Needed

Use the authentication field for protected API routes or admin areas. The field stays local to the browser unless you explicitly save it on your own device.

3

Use A Trusted Proxy

A relay proxy can help with CORS and missing response headers, but it can also see what your browser sends. Use only a proxy you operate or fully trust.

4

Export Reports, Not Secrets

Keep scan reports, discovered endpoints, and remediation notes. Do not hardcode private API keys, tokens, or backend credentials in static sites or public repositories.

Accuracy note: GitHub Pages cannot securely hide a real private API key. If you need protected integrations, move the secret handling to your own server-side function or relay.
